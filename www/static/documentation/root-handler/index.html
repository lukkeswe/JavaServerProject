Root Handler Flowchart

The Root Handler validates requests and return web related files.

flowchart TD A(Start) --> B(Get Headers and Host) B --> C{Host is null} C -->|Yes| D(Set host to unknown) C -->|No| E(Get request path and split) D --> E E --> F(Init htmlFilePath targetFile userPath) F --> G{userPath is null} G -->|Yes| H(Set default userPath notfound) G -->|No| I(Continue) H --> I I --> J{Path ends with html} J -->|Yes| K(Set targetFile) J -->|No| L{Path ends with php} L -->|Yes| M(Run PHP) L -->|No| N(Skip PHP) M --> O{PHP result not null} O -->|No| P(Return) O -->|Yes| Q(Set response) K --> R(Check Host type) Q --> R P --> Z(End) R --> S{Host is dev or ludwig} S -->|Yes| S1(Set htmlFilePath = userPath) S -->|No| S2(Check if path not root and not php) S1 --> T(Continue) S2 --> T T --> U(Handle static or irregular path) U --> V(Check file exists or fallback) V --> W{index.php exists} W -->|Yes| X(Run runPhp and send) W -->|No| Y(Send 403 or 404) X --> Z Y --> Z V --> AA(Check static extension CSS JS Image or fallback to index.html) AA --> AB(Serve or set content type then send) AB --> Z
